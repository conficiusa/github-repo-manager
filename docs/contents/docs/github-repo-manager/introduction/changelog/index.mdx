## Version 0.1.2 (Current)

**Release Date: 26/02/2025**

### New Improvements
- Fixed visibility parameter to correctly filter public/private repositories
- Improved error handling for authentication issues
- Enhanced terminal UI refresh performance

### Latest Bug Fixes
- Fixed pagination issue when navigating large repositories
- Corrected error message display on authentication failures
- Fixed memory usage issue with large repository lists

## Version 0.1.1

**Release Date: 26/02/202**

### New Features
- Added support for environment variables (`GITHUB_TOKEN` and `GITHUB_USERNAME`)
- Implemented keyboard shortcut 'a' to select all repositories
- Implemented keyboard shortcut 'n' to deselect all repositories

### UI Improvements
- Reduced API calls for better GitHub rate limit management
- Enhanced terminal UI with color-coding for better visibility
- Added confirmation count for selected repositories

### Previous Bug Fixes
- Fixed crash when no repositories were found
- Corrected display issues on smaller terminal windows

## Version 0.1.0

**Release Date: 26/02/2025**

### Initial Release Features
- Interactive terminal UI for repository selection
- Filter repositories by name pattern (regex support)
- Filter by visibility (public/private)
- Dry run mode for safe previews
- Confirmation steps to prevent accidental operations
- Interactive pagination for large repository lists
- Command-line interface with various options
- Python library interface for programmatic usage

## Future Plans

Features planned for upcoming releases:

- Support for GitHub Enterprise instances
- Bulk repository archiving operations
- Repository statistics and insights
- Support for organization-level operations
- Advanced filtering by repository age, activity, and other metrics
- Export functionality for repository metadata
